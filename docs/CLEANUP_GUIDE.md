# Old Files Cleanup Guide

These files are no longer needed after the WXT migration:

### Old HTML Pages (Replaced by entrypoints)
```
❌ pages/popup.html              → entrypoints/popup/index.html
❌ pages/options.html            → entrypoints/options/index.html
```

### Old Entry Scripts (Replaced by entrypoints)
```

❌ src/options/main.ts           → entrypoints/options/main.ts
❌ content.js                    → entrypoints/content/index.ts

```

❌ src/options/Options.vue       → entrypoints/options/App.vue

```

```

✅ scripts/package-xpi.js        → Keep (may be useful for custom packaging)

## 🧹 Cleanup Commands

### Safe Cleanup (Recommended)
```bash
# Create backup first
mkdir old_files_backup
mv pages old_files_backup/
mv src/popup.js old_files_backup/
mv src/AppPopup.vue old_files_backup/
mv src/options/Options.vue old_files_backup/
mv src/App.vue old_files_backup/
mv content.js old_files_backup/
mv manifest.json old_files_backup/
mv vite.config.ts old_files_backup/

# Test that everything works
npm run extension:dev

# If all works, remove backup
rm -rf old_files_backup
```

### Clean Build Artifacts
```bash
# Remove old build output
rm -rf dist/

# Remove node_modules (optional, then reinstall)
rm -rf node_modules/
npm install

# Clean WXT output
rm -rf .output/
```

### Git Cleanup
```bash
# Add old files to .gitignore if needed
echo "old_files_backup/" >> .gitignore
echo "dist/" >> .gitignore

# Commit the migration
git add .
git commit -m "Migrate to WXT framework

- Restructured project to follow WXT standards
- Moved entrypoints to entrypoints/ directory
- Updated configuration to use wxt.config.ts
- Removed old vite.config.ts and manifest.json
- Updated all npm scripts
- Fixed import paths"
```

## 📋 Migration Verification Checklist

Before removing old files, verify:

- [ ] Extension builds successfully: `npm run build:firefox`
- [ ] Extension runs in Firefox: `npm run extension:dev`
- [ ] Popup opens and works correctly
- [ ] Options page opens and works correctly
- [ ] Content script injects on target pages
- [ ] All Pinia stores work (globalStore, FinanseStore)
- [ ] Storage sync works between contexts
- [ ] All TypeScript types resolve: `npm run type-check`
- [ ] Tests pass: `npm run test:unit`
- [ ] No console errors in browser
- [ ] Hot reload works when editing files

## 🔄 Rollback Plan (If Needed)

If something breaks after migration:

1. **Restore from backup**:
   ```bash
   mv old_files_backup/* ./
   ```

2. **Revert package.json scripts**:
   ```bash
   git checkout HEAD~1 package.json
   npm install
   ```

3. **Use old build process**:
   ```bash
   npm run extension:build  # Old vite-based build
   ```
